[INST: What is the purpose of the "manual.md" file?]  The "manual.md" file is a detailed user manual that provides instructions on how to use the software, including introducing main functions of the software, how to install environment dependencies, and how to use/play it. It is intended to be a comprehensive guide for users who are new to the software or who want to learn more about its features and capabilities.

[INST: What is the purpose of the "requirements.txt" file?]  The "requirements.txt" file is a list of dependencies required to run the software. It specifies the versions of the dependencies that are required to run the software, and it is used by the package manager to install the dependencies. The file is typically created by the software developer and is included in the software distribution.

[INST: What is the purpose of the "main.py" file?]  The "main.py" file is the main entry point of the software. It is the file that is executed when the software is run, and it is responsible for setting up the environment, parsing the command-line arguments, and running the software. The file typically includes the main logic of the software, such as the code that interacts with the user, the code that interacts with the dependencies, and the code that implements the core functionality of the software.

[INST: What is the purpose of the "README.md" file?]  The "README.md" file is a file that provides information about the software, including a brief description of the software, its purpose, and how to use it. It is typically created by the software developer and is included in the software distribution. The file is intended to be a quick introduction to the software and to provide a starting point for users who are new to the software.
